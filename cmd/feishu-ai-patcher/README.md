# `cmd/feishu-ai-patcher`

## 模块职责

`cmd/feishu-ai-patcher` 是整个 Go 工具的可执行入口。

它的职责保持尽可能薄：

1. 调用 `internal/app` 创建应用实例。
2. 把命令行参数传给应用层。
3. 当应用层返回错误时，把错误打印到标准错误并以非零状态退出。

## 文件说明

### [`main.go`](./main.go)

程序入口文件：

- 调用 `app.New()` 构造应用。
- 传入 `os.Args[1:]` 启动主流程。
- 在失败时把错误输出到 `stderr`，让用户和外部调用方都能看到清晰的失败原因。

## 设计约束

`cmd` 层不放业务逻辑，不放规则，不放搜索算法，也不直接读写配置或备份文件。这样做的目的是：

- 保持入口足够简单，便于构建和分发。
- 把真正复杂的行为都留在 `internal/app` 和更底层模块中，便于测试和维护。
